How GitVault Works


                    

                        

                            

                                
                            

                            

                        

                        

                            

                                
                            

                            

                        

                        

                            

                                
                            

                            

                        

                        

                            

                                
                            

                        

                    

                    

                        Generate Key
                        Encrypt Files
                        Push to GitHub
                        Decrypt Locally
                    


                    

                        

                            
1

                            

                                
Generate an encryption key

                                
Create a new AES-256 key with one click. It stays in your browser's local storage — never touches any server. Export it as a .gitvault-key.json file for backup.

                            

                        

                        

                            
2

                            

                                
Encrypt sensitive files

                                
Drop your .env files, API keys, certificates, configs, or any sensitive files. They're encrypted with AES-256-GCM right in your browser — binary-safe, works with any file type.

                            

                        

                        

                            
3

                            

                                
Commit encrypted files to your public repo

                                
Replace the originals with their .encrypted versions and push. Anyone can see the files — nobody can read them without your key. Add a .gitvault config to document which files should be encrypted.

                            

                        

                        

                            
4

                            

                                
Share the key with teammates

                                
Export your key file and share it through a secure channel (encrypted message, in-person, password manager). Team members import the key to decrypt files locally. Never commit key files to the repo.

                            

                        

                        

                            
5

                            

                                
Decrypt when needed

                                
Pull the repo, drop encrypted files into the Decrypt tab, and get your originals back instantly. Everything happens locally — your decrypted files never leave your machine.

                            

                        

                    

                


                
                

                    
Security Guarantees

                    

                        

                            
Zero Knowledge

                            
All encryption runs in your browser using the Web Crypto API. No plaintext or keys ever leave your machine. There is no server to hack, subpoena, or compromise.

                        

                        

                            
AES-256-GCM

                            
Authenticated encryption with 256-bit keys. The same standard used by governments and banks. Includes integrity verification — tampered files are automatically rejected.

                        

                        

                            
No GPG Required

                            
Unlike git-crypt, git-secret, and BlackBox, GitVault uses simple JSON key files. No GPG keyrings, no trust models, no expired subkeys. Generate, export, import — that's it.

                        

                        

                            
Open Source

                            
Every line of code is auditable. No obfuscation, no tracking, no analytics. Verify that the encryption claims are real — don't trust, verify.

Use Cases

                    

                        

                            
Open Source Projects

                            
Keep API keys, deploy credentials, and config secrets encrypted alongside your public code. Contributors see the structure — only authorized team members see the values.

                        

                        

                            
Student Portfolios

                            
Showcase your projects publicly without exposing API keys, database URLs, or third-party credentials in your repository history.

                        

                        

                            
Small Team Collaboration

                            
Share one key file among your team. Everyone can encrypt/decrypt locally. No paid GitHub plans, no secret management infrastructure needed.

                        

                        

                            
GitOps & CI/CD

                            
Version your encrypted secrets alongside infrastructure code. Decrypt in your pipeline using the exported key file stored in your CI/CD secret store.
