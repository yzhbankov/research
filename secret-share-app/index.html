How SecretDrop Works


                    

                        

                            
1

                            

                                
You type your secret

                                
Enter any text — passwords, API keys, private messages, sensitive data.

                            

                        

                        

                            
2

                            

                                
Choose your key method

                                
Use a text passphrase, record a voice phrase, or draw a pattern. Your key never leaves your browser.

                            

                        

                        

                            
3

                            

                                
Encryption happens locally

                                
AES-256-GCM encryption runs entirely in your browser using the Web Crypto API. The server receives only ciphertext — it physically cannot read your data.

                            

                        

                        

                            
4

                            

                                
Share the link publicly

                                
Get a one-time link or QR code. Send it over any channel — email, Slack, SMS, carrier pigeon. Even if intercepted, it's useless without the key.

                            

                        

                        

                            
5

                            

                                
Recipient decrypts with the key

                                
They enter the passphrase (or voice phrase / pattern) and get the original message. Then it self-destructs.

                            

                        

                    


                    

                        
Security Guarantees

                        

                            

                                
Zero Knowledge

                                
All encryption/decryption uses the Web Crypto API in your browser. Your plaintext and key never touch our servers. We couldn't read your data even if compelled by law — because we don't have it.

                            

                            

                                
AES-256-GCM

                                
Military-grade authenticated encryption. The same standard used by governments and financial institutions worldwide. Includes integrity verification — tampered data is automatically rejected.

                            

                            

                                
PBKDF2 Key Derivation

                                
Your passphrase is strengthened with 100,000 rounds of PBKDF2-SHA256 and a random salt. This makes brute-force attacks computationally infeasible.

                            

                            

                                
Open Source

                                
Every line of code is auditable. Verify that the encryption claims are real — don't trust, verify.

                        How GitHub Vault Works
                        

                            

                                
1

                                

                                    
Generate an encryption key

                                    
Create a new AES-256 key. This key stays in your browser's local storage — it never touches any server.

                                

                            

                            

                                
2

                                

                                    
Encrypt sensitive files

                                    
Drop your .env files, configs, keys, or any sensitive files. They get encrypted with military-grade AES-256-GCM right in your browser.

                                

                            

                            

                                
3

                                

                                    
Commit encrypted files to your public repo

                                    
Replace the original files with their .encrypted versions and push. Anyone can see the files — nobody can read them without your key.

                                

                            

                            

                                
4

                                

                                    
Share the key with teammates

                                    
Export your key as a .secretdrop-key.json file and share it securely (use the Encrypt tab!). Team members import it to decrypt files locally.

                                

                            

                            

                                
5

                                

                                    
Decrypt when needed

                                    
Pull the repo, drop encrypted files into the Decrypt section, and get your original files back instantly.

                                

                            

                        

                        

                            Never commit your key file to the repository. The .secretdrop-key.json file should be added to your .gitignore. If compromised, generate a new key and re-encrypt all files.
